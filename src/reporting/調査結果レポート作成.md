## 画面別の追加検討項目一覧

### Reception
- **GAP-REC-01**: 右ペイン（患者概要/直近診療/オーダー概要）未実装 → 受付判断材料の不足
- **GAP-REC-02**: 一覧列（保険/自費/ORCA キュー/直近診療など）不足 → 状態判断の弱さ
- **GAP-REC-03**: 例外一覧（未承認/送信エラー/遅延）未整備 → 例外処理優先順位が不明
- **GAP-REC-04**: 監査履歴検索 UI がない → 追跡性/説明責任不足
- **GAP-REC-05**: 新規タブ導線未実装 → マルチ患者処理効率の低下

### Patients
- **GAP-PAT-01**: 監査履歴ビュー（詳細/検索/時系列）不足 → 監査説明責任不足
- **GAP-PAT-02**: ORCA 反映状態の表示がない → 編集確定性が不明
- **GAP-PAT-03**: 未紐付/警告の強調・通知がない → 受付/カルテへの影響共有が弱い

### Administration / 配信
- **GAP-ADM-01**: 管理メニュー全体設計未整備 → 管理対象の大半が未実装
- **GAP-ADM-02**: system_admin ガード（画面/サーバ両面）不十分 → 権限逸脱のリスク
- **GAP-ADM-03**: 配信タイミング/環境ラベル表示未整備 → 運用判断不可
- **GAP-ADM-04**: 管理変更の監査保持期間/検索 UI 未確定 → 監査要件が未確定
- **GAP-ADM-05**: 配信証跡（バージョン/ETag/反映遅延）の運用未確定 → 追跡性不足

### 認証/権限（共通）
- **GAP-AUTH-01**: 施設/ユーザ選択 UI がない → 複数施設運用の入口不足
- **GAP-AUTH-02**: 施設/ユーザ切替導線がない → 誤操作・権限越境リスク
- **GAP-AUTH-03**: 権限境界の運用フロー未整備 → 拒否理由/承認導線が不足

### Charts（カルテ入力以外）
- **GAP-CHT-01**: 印刷の実 PDF 生成/プレビュー/復旧導線が限定的 → 印刷業務が未完結
- **GAP-CHT-02**: 診療終了/送信の実処理未整備 → 受付・会計連携が未完
- **GAP-CHT-03**: 患者安全ヘッダ未実装 → 誤認防止導線不足
- **GAP-CHT-04**: ドキュメント履歴（検索/フィルタ/ガード）未充足 → 履歴確認と監査が弱い

### 共通
- **GAP-COM-01**: RUN_ID コピー/通知トースト/ナビバッジ未実装 → 障害時運用支援不足
- **GAP-COM-02**: data-run-id/aria-live の統一不足 → 監査・可観測性の整合不足

- **ORCA 実機検証の未確定**: 送信/印刷/ORCA 反映の実運用証跡が不足。
- **監査ログ粒度と検索基盤**: 監査イベントの必須メタ/保存/検索方針が未合意。

- **配信運用の説明責任**: 即時/遅延/再ログインの基準が曖昧なまま UI を作ると手戻りリスク。
- **性能影響**: 監査ログ増加と検索 UI の追加でパフォーマンス劣化の評価が必要。
