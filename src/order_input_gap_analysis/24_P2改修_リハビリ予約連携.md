# 24 P2 改修：リハビリ予約/スケジュール連携

RUN_ID: 20251229T081044Z

- 目的: リハビリ予約/スケジュール連携 UI の可否を判断し、代替運用を整理する。
- 対象: リハビリオーダー入力（Charts 右パネル）。
- 実装観点:
  - 予約/スケジュール API の有無と利用可否を調査。
  - 実装不可の場合は代替運用（手入力/別システム）を明文化。
  - 実装可能なら UI 導線と監査ログ方針を定義。
- 依存/前提: 予約/スケジュール連携仕様の確認（不足時は server-modernized タスク起票）。
- テスト: 代替運用の入力誘導文言と UI 表示確認。
