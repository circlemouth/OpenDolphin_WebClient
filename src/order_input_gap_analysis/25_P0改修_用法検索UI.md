# 25 P0 改修：用法検索/フィルタ UI

RUN_ID: 20251229T213910Z

- 目的: 処方オーダーで旧来版同等の用法検索/フィルタ UI を提供する。
- 対象: OrderBundleEditPanel（処方/RP の用法入力）。
- 実装観点:
  - 用法マスタ検索（部分一致/件数制限）と結果選択の導線を追加する。
  - 検索結果選択で用法コード/表示名を一括反映する。
  - readOnly/missingMaster/fallbackUsed ガードと競合しない導線を確保する。
- 依存/前提: `/orca/master/youhou` API の検索仕様（keyword/effective）に従う。
- テスト: 用法検索→選択→保存、入力ブロック時の UI 挙動。

## UI 仕様（Web）
- 用法検索欄:
  - キーワード入力 + 用法フィルタ（内服1回/2回/3回/4回、点眼、塗布、頓用、吸入、全て）。
  - 部分一致チェック（入力値を含む名称/コードで絞り込み）。
  - 件数上限（20/50/100）で表示件数を制御。
- 検索結果:
  - コード/名称/単位/分類/備考を一覧化し、行選択で用法へ反映。
- 反映仕様:
  - `admin` に `code + name` をセット。

## 実装メモ
- OrderBundleEditPanel に用法検索セクションを追加。
- `readOnly/missingMaster/fallbackUsed` で入力を無効化。
- `fetchOrderMasterSearch(type="youhou")` を利用し、空キーワード時はフィルタ選択時のみ許可。

## テスト観点
- 用法検索結果の選択で `用法` 入力へ反映されること。
- readOnly 時に用法検索 UI が無効化されること。
