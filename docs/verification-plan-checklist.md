# 確認作業計画テンプレート挿入用チェックリスト

> 用途: 不足/要確認/リスク対策の短文案をチェックリスト形式に整理する。
> 差し込み位置案: 「5.1 チェックリスト」に追加、必要に応じて「7. リスクと対応」に転記。

## 1. 不足しやすい観点（画面/サーバー）

| 観点 | チェック項目 | Yes | No | 要確認 | 備考 |
| --- | --- | --- | --- | --- | --- |
| 画面 | 状態別表示（空/読み込み中/エラー/権限なし）が定義されている |  |  |  |  |
| 画面 | 権限差分（閲覧/編集/管理者）の表示・操作が整理されている |  |  |  |  |
| 画面 | エラー文言が具体的で再現/解消の手がかりを示す |  |  |  |  |
| 画面 | 遷移/リロードで状態が破綻しない |  |  |  |  |
| 画面 | 検索/一覧の組合せ（並び替え/絞り込み/ページング）が確認対象 |  |  |  |  |
| 画面 | アクセシビリティ（キーボード/フォーカス/コントラスト）が確認対象 |  |  |  |  |
| 画面 | ファイルアップロードの失敗系（容量/拡張子/途中失敗）が確認対象 |  |  |  |  |
| サーバー | APIエラー分類（4xx/5xx/タイムアウト）とリトライ方針が定義されている |  |  |  |  |
| サーバー | 認可/スコープの差分（ロール別制限）が確認対象 |  |  |  |  |
| サーバー | 競合/排他（同時更新/重複登録）のシナリオが確認対象 |  |  |  |  |

| サーバー | バッチ再実行条件と影響範囲が明確 |  |  |  |  |
| サーバー | 監査ログの取得項目・保存期間が明確 |  |  |  |  |
| サーバー | 監視・アラートの閾値と一次対応フローが明確 |  |  |  |  |

## 2. 要確認事項（短文チェック）

| 観点 | チェック項目 | Yes | No | 要確認 | 備考 |
| --- | --- | --- | --- | --- | --- |
| 画面 | 入力バリデーション（必須/任意、境界値、文字種）が揃っている |  |  |  |  |
| 画面 | 画面遷移（ブラウザバック/リロード）で状態が破綻しない |  |  |  |  |
| 画面 | ロール/権限別の表示・操作差分が確認できる |  |  |  |  |
| 画面 | アクセシビリティ観点（キーボード/フォーカス/コントラスト）がある |  |  |  |  |
| サーバー | 外部連携失敗時の挙動（再送/保留/通知）が確認されている |  |  |  |  |
| サーバー | ログ/監査の取得項目・保存期間が明確 |  |  |  |  |
| サーバー | 設定変更の反映タイミングとロールバック手順がある |  |  |  |  |

## 3. リスクと対策（チェックリスト化）

| リスク | 対策 | Yes | No | 要確認 | 備考 |
| --- | --- | --- | --- | --- | --- |
| 状態別表示の未確認 | 状態一覧を洗い出し必須確認に追加 |  |  |  |  |
| 権限差分の未確認 | ロール別チェックリストを作成 |  |  |  |  |
| エラー文言の不備 | 文言レビューを作業に組み込む |  |  |  |  |
| 外部連携障害時の未確認 | 失敗時挙動を仕様化し確認 |  |  |  |  |
| 同時更新の未確認 | 競合シナリオを追加 |  |  |  |  |
| バッチ再実行の未確認 | 再実行手順とガードを明記 |  |  |  |  |
| 監査ログの未確認 | ログ項目一覧を別表で作成 |  |  |  |  |
| キャッシュ反映の未確認 | 反映タイミング確認を追加 |  |  |  |  |
