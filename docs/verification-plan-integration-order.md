# 確認作業計画 最終差し込み順序（優先度と依存関係）

> 目的: `docs/verification-plan.md` の残プレースホルダを、依存関係に基づいて効率的に差し込む。
> 参照: `docs/verification-plan-placeholder-map.md`

## 1. 差し込み順序（優先度順）

1. 画面一覧（`## 4` / `### 4.1`）
2. サーバー機能一覧（`## 5` / `### 5.1`）
3. トレーサビリティ表（`## 6` / `### 6.1`）
4. 画面レビュー本文案（`## 4` / `### 4.2` 以降）
5. 要確認事項の追記（`## 10`）
6. チェックリストの更新（`## 8` / `### 8.1`）

## 2. 依存関係

- トレーサビリティ表は「画面一覧」と「サーバー機能一覧」の確定が前提。
- 画面レビュー本文案は「画面一覧」のID/名称に依存。
- 要確認事項・チェックリストは独立だが、画面/機能一覧の確定後に最終調整が望ましい。

## 3. 作業手順（箇条書き）

- 画面一覧表を `### 4.1 画面一覧` に差し込む（ID形式は `SCR-xxx`）。
- サーバー機能一覧草案を `### 5.1 サーバー機能一覧フォーマット` に差し込む（ID形式は `SVR-xxx`）。
- 画面×機能の対応表を `### 6.1 トレーサビリティ表フォーマット` に差し込む（依存区分は 主要/補助/参照）。
- 画面レビュー本文案を `### 4.2 画面ごとの確認観点フォーマット` 配下に追記し、フォーマットを維持。
- 要確認事項の追加があれば `## 10` に追記する。
- チェックリストの更新があれば `### 8.1` の表を置換/追記する。

## 4. 最終確認

- すべての [PLACEHOLDER] が解消されているか確認。
- 見出し構成と章番号が崩れていないか確認。
- 変更履歴に反映日と内容を記載。
