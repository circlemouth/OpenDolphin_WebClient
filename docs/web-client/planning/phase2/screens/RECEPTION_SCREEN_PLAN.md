# 受付状況処理画面 設計ドラフト（RUN_ID=`20251130T120000Z`）
電子カルテ全体の中で、受付管理画面は
「今日の患者の流れを見える化して、カルテ画面への入口になるハブ」
という位置づけになります。

⸻

1. 受付管理画面の役割・前提

前提として、この画面でやりたいことは：
	1.	今日の患者の
	•	受付状況
	•	診察の進み具合
	•	診察終了後の会計連携状況（ORCA連携）
を一目で把握する
	2.	そこから 個別カルテ画面（内部タブ方式）を開く
	3.	必要に応じて
	•	基本情報（住所・性別など）の編集
	•	自費／保険の区別
	•	簡易な診療履歴・メモの参照

です。

⸻

2. 画面全体のレイアウト

2-1. 上部：共通ヘッダー

ヘッダーに置きたいもの
	•	施設名／ロゴ
	•	現在日付・時間（「本日の受付」だと分かる）
	•	ログインユーザー名＋ロール
	•	例：山田 太郎（受付）、佐藤 花子（医師）
	•	グローバルメニュー
	•	「受付管理」「カルテ一覧」「管理者画面」などへのナビゲーション
	•	簡単なシステムメッセージ／アラート表示
	•	例：ORCA連携エラーがある場合の通知

⸻

2-2. 左：ステータス別のタブ／フィルタ

ここが「受付管理画面の要」です。
少なくとも、ここまでの話から必須なのは以下の2タブ：
	1.	受付状況タブ
	•	「受付済だけど診察が終わっていない患者」の一覧
	•	＝「診察待ち・診察中」を全部ここで見るイメージ
	2.	診察終了タブ
	•	診察が終わった患者
	•	ORCAへオーダー／会計情報を送る対象

ここに、必要であればオプションで細分化してもよいです：
	•	受付待ち / 診察待ち / 診察中
	•	診察終了(ORCA未送信) / 会計送信中 / 会計済み

タブ共通のフィルタ
各タブの上部に、以下のようなフィルタを付けると運用が楽です：
	•	診療科：内科／小児科／整形…など
	•	担当医師：○○先生の患者だけ表示
	•	受付時間帯：午前／午後
	•	自費診療かどうかのフィルタ（自費フラグ）

⸻

2-3. 中央：患者一覧テーブル

選択中のタブ（受付状況／診察終了）に応じて、
中央に患者の一覧テーブルを表示します。

一覧に欲しい列（受付状況タブ）
	•	ステータス
	•	受付済／診察待ち／診察中 など
	•	患者ID
	•	患者氏名（＋ふりがな）
	•	年齢・性別
	•	自費／保険区別アイコン
	•	受付時間
	•	待ち時間（受付時刻からの経過）
	•	予約区分（予約／飛び込み）
	•	診療科
	•	担当予定医師
	•	注意フラグ（メモがあるかどうかを示すアイコン）

行ごとの操作ボタン
	•	「カルテを開く」
→ ブラウザ内の疑似タブで、個別カルテ画面を開く
	•	「基本情報編集」
→ 住所・電話・保険情報の編集画面へ遷移
	•	「受付修正」
→ 担当医・診療科・予約種別などの変更
	•	「受付取消」
→ 間違って登録した場合のキャンセル（ログは残す）

診察終了タブの一覧列
診察終了タブは、ORCA連携までを見届けるテーブルになります。
	•	患者ID／氏名／年齢・性別
	•	診療科・担当医
	•	診察終了時刻
	•	会計連携ステータス
	•	未送信／送信中／送信完了／エラー
	•	保険／自費区別
	•	最終病名の有無（保険病名が未入力の場合に警告）

行ごとの操作ボタン
	•	「カルテ再表示」
→ 診療内容の再確認や修正のためにカルテタブを開く
	•	「ORCA送信」または「再送信」ボタン
	•	「会計完了にする」チェック（ORCA側で完了確認後に使う）

⸻

2-4. 右：選択患者の詳細パネル

中央の一覧で1行を選択すると、
右側に「クイックビュー」を表示すると便利です。

ここに表示したいもの（ここまでの話を反映）
	1.	基本情報サマリ
	•	氏名／カナ
	•	生年月日・年齢・性別
	•	住所・電話番号
	•	保険情報（保険者番号などの一部サマリ）
	•	自費／保険の区別
	2.	簡易診療履歴
	•	前回〜数回分の診療日・診療科・担当医
	•	ざっくりした病名／サマリ（「風邪」「高血圧フォロー」など）
	•	「詳細をカルテで開く」リンク
	3.	処方・オーダー履歴（概要）
	•	最近よく出している薬の一覧（薬品名だけ）
	•	主な検査・処置の履歴（「採血」「レントゲン胸部」程度の粒度）
	•	「詳細履歴」を開くリンク（カルテ側に飛ぶ）
	4.	患者ごとのメモ
	•	受付／診察両方で使える「ちょっとした注意事項」
	•	例：
	•	聴力が弱い
	•	車いす
	•	会計時に説明が必要な事項 など
	•	編集ボタン：受付でも書き足せるようにするかどうかは権限で制御

⸻

3. 受付管理画面で必要な機能一覧

3-1. 患者受付登録・修正機能

新規受付登録
	•	既存患者の検索 → 選択して受付
	•	氏名／フリガナ／生年月日などで検索
	•	新患の場合は、簡易登録 → 基本情報編集画面へ誘導
	•	入力項目例
	•	診療日（通常は今日固定）
	•	診療科
	•	担当医（未定も可）
	•	予約／飛び込み区別
	•	自費／保険フラグ

受付内容の修正
	•	担当医変更
	•	診療科変更
	•	予約枠変更
	•	受付取消（キャンセル）

⸻

3-2. 基本情報編集への導線

ここまでの話から、
患者の住所・性別など基本情報を編集する画面が別途必要、
という前提があるので：
	•	受付一覧の行に「基本情報」ボタンを用意
	•	右側クイックビューの基本情報サマリの中に「編集」リンク

この編集画面では：
	•	氏名／カナ
	•	生年月日・性別
	•	住所・電話番号
	•	保険情報（保険証番号、公費情報など）
	•	緊急連絡先

を編集し、保存後に受付画面へ戻る動線を想定。

⸻

3-3. 自費診療／保険診療モードの扱い

ユーザーからの要望：

患者さんの個別のカルテを開いて、
保険診療カルテと自費診療カルテを切り替えられるようにする

これを受付画面では、少なくとも次の2点で扱います。
	1.	受付時点でのモード指定
	•	受付登録時に「保険」「自費」「併用」のいずれかを選択
	•	一覧テーブルにもアイコン表示（例：￥フリーのマーク、自費マーク）
	2.	カルテ画面への引き継ぎ
	•	「カルテを開く」を押したときに
受付で指定したモードをカルテ側に渡して
デフォルトで保険カルテ or 自費カルテを開く

⸻

3-4. 診療ステータスの管理（受付 → 診察 → 診察終了）

受付管理画面上でのステータス遷移イメージ：
	1.	受付済（診察待ち）
	•	登録した時点の状態
	2.	診察中
	•	医師がカルテを開いたタイミング／診察開始ボタンで切り替え
	3.	診察終了
	•	医師がカルテ上で「診療終了」操作をしたらこの状態に遷移
	•	→ 自動的に「診察終了タブ」側へ移動

受付管理画面の役割としては、
	•	状態を色やアイコンで分かりやすく表示
	•	必要なら受付側からステータスを手動調整（誤操作時の修正用）

⸻

3-5. ORCA（WebORCA）との連携状態表示

診察終了後、
ORCAに会計情報・オーダー情報を飛ばすという前提なので、
診察終了タブでは、以下が必要になります。
	•	ORCA送信ステータス
	•	未送信
	•	送信中
	•	送信完了
	•	エラー（エラー理由の簡易表示）
	•	ORCA送信用の操作
	•	「送信」ボタン（自動送信＋手動再送の両方を想定）
	•	エラー時に詳細ログを確認するリンク（管理者用）

受付管理画面は「医事との境目」なので、
ここで状態を可視化しておくと、受付・医事・医師の連携がしやすくなります。

⸻

3-6. 簡易な診療履歴・処方履歴の参照

最初に出てきた要望：

受付画面で簡易的に患者さんの診療歴を見れるようにしたい

これを実現するためには：
	•	右側の詳細パネルに
	•	最終診察日
	•	最近の主病名
	•	最近出している薬の一覧（数件）
	•	最近の主なオーダー（検査・処置）

を表示する。

ポイント
	•	あくまでも「ざっくり確認用」なので、詳細はカルテ画面に任せる
	•	受付でも「この方はいつも●●の薬をもらっている」程度が分かると会話がスムーズ

⸻

3-7. 患者メモの取り扱い

ユーザーの要望：

アラートとリマインダーはいらないけど、患者さんごとのメモは見れた方がいい

受付画面での振る舞い：
	•	右側の詳細パネルに「患者メモ」欄
	•	一覧テーブルにも「メモあり」アイコン
	•	受付スタッフでも追記して良いメモかどうかは権限で制御
	•	例：受付専用メモ／医師・看護師専用メモを分けるのもアリ

⸻

3-8. 検索・ソート・その他ユーティリティ

検索
	•	今日以外の過去受診歴から患者を検索して
今日の受付に追加するケースも想定されるので、
患者全体検索ダイアログがあると便利です。

ソート
	•	待ち時間でソート（長い順）
	•	診療科／担当医別ソート
	•	受付時間順

その他
	•	画面の自動更新（一定間隔でリストを更新）
	•	印刷／エクスポート（受付簿として出す場合）

⸻

3-9. 権限・ログの観点（受付画面に関係する部分）

ユーザー管理全体の話の中から、
受付画面に直接関係するものを抜き出すと：
	•	ロールごとの制御
	•	受付：受付登録・修正・一部基本情報編集はOK
	•	看護師：受付リスト閲覧＋患者呼出しはOK、受付内容の変更は制限
	•	医師：主に閲覧＋カルテを開く
	•	操作ログ
	•	誰が
	•	いつ
	•	どの患者の
	•	受付内容をどう変更したか
を記録（監査・トラブル時のため）

⸻

まとめ

ここまでの会話をベースにすると、
受付管理画面に必須なのは：
	•	ステータス別タブ（受付状況／診察終了）
	•	患者一覧テーブル（ステータス、基本情報、自費／保険、受付時間など）
	•	個別カルテ画面への入口（「カルテを開く」）
	•	基本情報編集への導線
	•	自費／保険フラグの管理とカルテへの引き継ぎ
	•	診療ステータスの表示とORCA送信ステータスの表示
	•	簡易な診療履歴・処方履歴・患者メモのクイックビュー
	•	検索・フィルタ・ソートなどの一覧操作
	•	ロールごとの権限と操作ログ

このあたりを押さえておけば、
実際のワークフロー「受付 → 診察 → 診察終了 → ORCA連携」を
受付側からストレスなく回せる構成になると思います。
