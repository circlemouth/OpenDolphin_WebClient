## 1. 実施手順
1. `web-client` で上記環境変数を付けて Vite dev（https://localhost:4173）を起動し、`/tmp/vite-stage.log` に起動ログを保存。
2. Playwright CLI (`npx playwright screenshot https://localhost:4173`) で Reception 画面をキャプチャし、上記スクリーンショットパスに出力。
3. stage.open-dolphin への接続性を `curl -I http://stage.open-dolphin` で確認 ― 「Could not resolve host」で DNS 解決自体が失敗したため、Stage サーバーからの JSON 取得が一切行えなかった。詳細は補助ログへ記録した。 

## 2. 観察結果
- Stage proxy/TCP 自体が `stage.open-dolphin` を解決できず、Reception/Charts は MSW ではなくクライアント起点の接続先未指定バナー（`dataSourceTransition` 未取得）を表示していた。
- `dataSourceTransition=server` や `missingMaster` バナーの表示タイミングを Stage 実サーバーで確認できない状態なので、当該フローは再現できず、Playwright/Charts の `ToneBanner` も server フラグに切り替わらなかった。

## 3. 次のアクション
- `stage.open-dolphin` の DNS（/ネットワーク）が復旧した段階で同一 RUN_ID を再使用し、再度 Reception→Charts を走らせて `dataSourceTransition=server`/`missingMaster` を観測・アーティファクト化。
- DOC_STATUS 及び UX 計画では本ログとスクリーンショットを RUN_ID として参照し、段階的な改善履歴を保持。
