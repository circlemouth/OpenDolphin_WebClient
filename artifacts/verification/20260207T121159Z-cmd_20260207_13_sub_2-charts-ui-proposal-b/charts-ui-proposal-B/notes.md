# cmd_20260207_13_sub_2: Charts UI改善案B（提案のみ / UIのみ）

RUN_ID: 20260207T121159Z-cmd_20260207_13_sub_2-charts-ui-proposal-b

## 現状課題（5-10行）
- 3カラム + ユーティリティで視線移動が大きく、SOAP（中央）が主役に見えにくい。
- カラム見出し・罫線・カード影が多く、視覚ノイズで情報の優先度が埋もれる。
- 画面余白/ギャップが大きく、ファーストビューで見える実作業領域（SOAP/タイムライン）が減る。
- 中央（SOAP）と他カードの見た目の差が小さく、「今どこを触るべきか」が直感で分かりづらい。
- ユーティリティがコンパクト幅の時でも説明文の密度が高く、狭幅で読めず視認性が下がる。
- カラム間の境界線が強く、縦割り感が出て「ひとつの作業台」としての連続性が弱い。
- カード毎の境界/背景が均一で、重要箇所（SOAP入力）への誘導が弱い。
- 重要な状態（RUN_ID/監査/送信）の情報はあるが、見た目のまとまりが薄く把握に時間がかかる。

## 改善案B（UIのみ / 機能変更なし）
- Center-first 配分: 3カラムは維持しつつ、中央を広め・左右をやや絞って「SOAP主作業」を強調。
- 余白/ギャップの最適化: ページpadding・カード間gapを少し詰め、ファーストビューの実作業面積を増やす。
- カラム境界の弱め: 罫線のコントラストを落として縦割り感を緩和。
- カラム見出しの軽量化: 見出しをコンパクトなセクションラベル風にし、情報は保持しつつノイズを削る。
- カードchromeの抑制: 影を抑え、境界線を薄めて可読性と集中度を上げる。
- SOAPの視覚的主役化: SOAPカードに薄いティント/アクセント枠を入れ、現在作業の「中心」を視認できるようにする。
- ユーティリティ（compact時）の情報量調整: `data-utility-state='compact'` ではショートカット一覧を非表示にして、狭幅での圧迫感を減らす（機能は維持、表示のみ調整）。

## UIのみ根拠（機能変更に該当しない理由）
- いずれも `chartsStyles` のCSS変数・ボーダー/影/余白/タイポグラフィ調整で実現可能（API/データ/業務ロジック不変）。
- カラム構成（左/中央/右/ユーティリティ）は維持し、表示優先度と視覚階層のみを変更。
- SOAP強調は背景色・枠・影の追加のみで、入力項目/保存/送信などの挙動は不変。
- ユーティリティのショートカット非表示は compact 状態での表示調整のみ（操作導線や機能自体は変更しない）。
