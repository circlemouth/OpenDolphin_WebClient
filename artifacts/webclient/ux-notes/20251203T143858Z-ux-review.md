## 背景
- docs/web-client/ux/ux-documentation-plan.md を起点に reception/patients/admin/charts の草稿を再読し、外来カルテと受付に限定した tone・レイアウト・ARIA/監査要件を整理。
- Legacy 系資料 (`docs/web-client/ux/legacy/`) は履歴参照のみとし、入院向け ORCA API やバナーは本スコープから除外した。

## 観察
### Reception
- トーン: 今日の患者フローを安心感のある一覧で可視化し、ステータス別フィルタや待ち時間の視覚重み（色・アイコン）で瞬時に状況把握。エラーバナー（ORCA 送信/未紐付/遅延）は赤/琥珀/青で統一し、`role=alert` + `aria-live=assertive` で即時読み上げ。完了/情報は `aria-live=polite` + `aria-atomic=false` で再読み上げを抑制し、`data-run-id` で Charts との更新同期を支援。
- レイアウト比率: 左レール（ステータスタブ＋フィルタ）→中央テーブル（患者情報＋オペレーション）→右パネル（基本情報サマリ＋処方/検査の概要）で 3 カラム構成。中央テーブルが主役で、右パネルは詳細補完、左レールは操作性サポート。
- 監査/ログ: 一覧自動更新/手動更新/ステータス変更/ORCA 再送ごとに `action/patientId/queueStatus/tone/ariaLive/runId` を記録し、Playwright の `fetchAuditLog` 検証にも利用。

### Charts（Chart Entry + Claim）
- ヘッダーに患者＋受付状態＋保険/自費トグルを常設し、中央のタブ群（診療録・病名・オーダー・結果・画像・文書・サマリ）をナビゲート。右サイドバーに患者メモ・未紐付チェック・ORCA/病名候補などを固定表示。
- バナーは Reception と同文言・トーンで `aria-live=assertive`（遅延/エラー）・`aria-live=polite`（完了）を共有し、診療終了＋署名で ORCA 送信をトリガー。Charts で表示されたバナーは Reception の診察終了タブへ carry over する。
- ステータス遷移/自動更新/再送は監査ログへの記録を義務付け、`manualRefresh` フラグや `aria-live` のトーンを保持しつつ 30 秒間隔の polling を続ける。

### Patients/Administration
- Patients 管理は Reception の「基本情報編集」からディープリンクされ、検索/新患/保険/メモ編集後は元のタブ・フィルタ・保険モードの状態を維持したまま戻す。履歴リンクと保険/自費モード保持（ローカルストレージ＋クエリ）を確認し、権限不足時は同様に戻す。
- Administration は左メニュー＋右詳細フォームで ORCA 設定・ステータス/ロール/テンプレなどを管理。role=system_admin/管理者のみアクセス可とし、更新タイミングは即時配信 or 次回リロードいずれかを明示。遅延時は Reception/Charts でバナー＋再取得/再ログイン導線を案内。
